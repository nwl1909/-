# ♠ Дурак онлайн

Классическая карточная игра «Дурак» (подкидной, 36 карт) прямо в браузере.

Два режима:
- **`index.html`** — игра против компьютера. Работает сразу, без всякой настройки.
- **`multiplayer.html`** — игра вдвоём с другом онлайн через комнату с кодом. Требует один раз настроить бесплатный проект Supabase (10 минут, см. ниже).

**[Играть →](https://YOUR_USERNAME.github.io/YOUR_REPO/)** *(ссылка появится после публикации через GitHub Pages)*

---

## Как запустить локально

Просто откройте `index.html` в браузере — двойным кликом или через `open index.html` / `start index.html`.

Для `multiplayer.html` желательно раздавать файлы через локальный сервер (не строго обязательно, но некоторые браузеры капризничают с `file://` и внешними скриптами):

```bash
python3 -m http.server 8000
# затем откройте http://localhost:8000/multiplayer.html
```

## Публикация на GitHub Pages

1. Создайте репозиторий на GitHub и загрузите в него все файлы: `index.html`, `multiplayer.html`, `multiplayer.js`, `config.js`, `README.md`.
2. **Settings → Pages → Build and deployment → Source: Deploy from a branch.**
3. Branch `main`, папка `/ (root)`, **Save**.
4. Через минуту-две сайт будет на `https://<логин>.github.io/<репозиторий>/`.

Сборка не нужна — GitHub Pages отдаёт файлы как есть.

## Настройка мультиплеера (Supabase)

Мультиплеер работает через [Supabase](https://supabase.com) — бесплатный облачный Postgres с realtime-обновлениями. GitHub Pages сам по себе не может держать соединение между двумя игроками (это просто хостинг статичных файлов), поэтому нужен внешний бесплатный бэкенд. Supabase выбран, потому что всё подключение — это пара строк JS в `config.js`, свой сервер поднимать не надо.

### Шаг 1 — создайте проект

1. Зайдите на [supabase.com](https://supabase.com), зарегистрируйтесь (можно через GitHub).
2. **New Project** → придумайте имя, задайте пароль базы данных (сохраните его куда-нибудь, хотя для этой игры он не понадобится напрямую), выберите ближайший регион.
3. Подождите 1-2 минуты, пока проект поднимется.

### Шаг 2 — создайте таблицу

1. Слева в меню откройте **SQL Editor**.
2. Откройте файл `supabase_setup.sql` из этого репозитория, скопируйте всё содержимое.
3. Вставьте в редактор и нажмите **Run**.

Это создаст таблицу `durak_games`, включит для неё Row Level Security с открытыми политиками (подходит для казуальной игры между друзьями по коду комнаты — подробности в комментариях самого SQL-файла) и включит Realtime.

### Шаг 3 — впишите ключи в config.js

1. В Supabase слева откройте **Project Settings → API**.
2. Скопируйте **Project URL** и **anon public** ключ.
3. Откройте `config.js` в репозитории и впишите оба значения:

```js
const SUPABASE_URL = "https://abcdefghijk.supabase.co";
const SUPABASE_ANON_KEY = "eyJhbGciOi...ваш длинный ключ...";
```

4. Сохраните, закоммитьте и запушьте — на GitHub Pages сайт сам подхватит новый `config.js`.

Это всё! `anon`-ключ безопасно публиковать в коде на GitHub — это ожидаемая практика Supabase, доступ ограничивается политиками RLS, которые задаёт `supabase_setup.sql`.

### Как это работает

- Один игрок нажимает «Создать комнату» — получает 6-значный код (например `K7X2QP`) и видит его на экране ожидания.
- Игрок отправляет код другу любым способом (мессенджер, голосом — как угодно).
- Второй игрок вводит код в поле «Присоединиться» — игра начинается автоматически у обоих.
- Все ходы синхронизируются через Supabase Realtime: как только один игрок делает ход, у второго это мгновенно (обычно <1 секунды) отражается на экране.
- Партия хранится в базе, так что можно закрыть вкладку и вернуться позже по тому же коду — игра продолжится с того места, где остановились.

### Важно про приватность

Это казуальный вариант без строгой защиты от читерства: оба игрока технически могут прочитать всё состояние партии (включая карты соперника) через API, если специально полезут в devtools. Для игры с другом это не проблема — это не более "уязвимо", чем подглядывание в карты за столом. Если понадобится защита понадёжнее (скрытие чужой руки на уровне сервера), это отдельная задача — дайте знать.

## Правила игры

- Колода — 36 карт (от 6 до туза), козырь определяется случайной картой при раздаче.
- Каждому — по 6 карт. Ходит первым тот, у кого младший козырь.
- Атакующий кладёт карту, защищающийся бьёт её старшей картой той же масти или козырем.
- Пока защищающийся не отбился, можно подкидывать карты того же достоинства, что уже на столе (не больше карт, чем осталось у защищающегося на руках).
- Отбился — карты уходят в «бито», ход атаки переходит к защищавшемуся.
- Не отбился — забирает все карты со стола себе, атака переходит к следующему.
- После розыгрыша игроки добирают карты из колоды до 6, начиная с атаковавшего.
- Цель — избавиться от всех карт первым. Оставшийся с картами, когда колода и рука соперника пусты — «дурак».

Полное описание правил доступно прямо в игре по кнопке **«?»**.

## Технологии

- Чистый HTML/CSS/JS — без React, без сборки, без npm-зависимостей
- Мультиплеер: [Supabase](https://supabase.com) (Postgres + Realtime) через `@supabase/supabase-js` с CDN
- Шрифты: Playfair Display + Inter (Google Fonts)

## Структура репозитория

```
.
├── index.html          — игра против компьютера (работает сразу)
├── multiplayer.html     — игра с другом онлайн (нужна настройка Supabase)
├── multiplayer.js        — логика мультиплеера и правила игры
├── config.js             — сюда вписываются ваши ключи Supabase
├── supabase_setup.sql    — SQL-скрипт для настройки базы данных
└── README.md             — этот файл
```
